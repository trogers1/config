# `.lib` definition

Directories named `*.lib/` are self-contained libraries with an intentionally
small public API.

## Rules

### Public imports (from outside the lib)

Only import from a `.lib` directory through its public `index.ts` entrypoint.

- Allowed: `import { x } from "./foo.lib"` or `import { x } from "./foo.lib/index"`
- Not allowed: `import { x } from "./foo.lib/internalThing"`
- Not allowed: `import { x } from "./foo.lib/some/nested/file"`

### Internal imports (from inside the lib)

Files inside a `.lib` directory must import sibling modules directly, not
through that lib's `index.ts`.

- Allowed: `import { x } from "./internalThing"` (when you are inside `foo.lib/`)
- Not allowed: `import { x } from "./index"` (when you are inside `foo.lib/`)
- Not allowed: `import { x } from "../foo.lib"` (when you are inside `foo.lib/`)

Exception: `index.test.ts` may import `./index` — the public entrypoint itself
stays unit-testable.

This keeps `index.ts` as a true public boundary. If an export is only used
inside the lib, do not add it to `index.ts`; knip will flag unused public
exports once internal index imports are removed.

## Why this exists

- Prevents accidental expansion of a library's public surface area
- Keeps library internals private and easier to change
- Limits dependency sprawl across the repo
- Preserves clear architecture boundaries between self-contained libs

## If you need something that is not exported

Choose one of these:

1. Export it from that lib's `index.ts` if it should be public
2. Move the shared code to a better public boundary
3. Reconsider whether the caller should depend on that internal detail at all

## Enforcement

Enforced by dependency-cruiser (`npm run check:deps`, part of `check:all` and
`pretest`) via rules `lib-public-entrypoint-only` and
`lib-no-index-self-import` in `.dependency-cruiser.cjs`:

- Deep imports into `*.lib/` from outside the lib are blocked.
- Imports of a lib `index.ts` from inside a lib are blocked. This is exact
  while a single `*.lib/` exists; when a second one appears, replace the
  generic rules with per-lib rules so cross-lib imports through the public
  index stay legal (dependency-cruiser cannot correlate from/to paths — see
  the rule comments).

knip (`npm run check:knip`) flags public exports nothing outside uses — remove
them from the index rather than importing through the index from inside the
lib.

## Adoption

Going forward: new self-contained module directories use the `.lib` suffix
(first: `modules/sandbox.lib/`, see `plans/sandbox_plan.md`). Existing flat
modules (`modules/policy.ts`, `modules/shell/`) predate the convention and may
be migrated opportunistically by adding an `index.ts` and updating import
sites.
